# Auto-Learn 使用说明

本文面向 Quorum 用户，说明 Auto-Learn 的目标、适用方式，以及在 TUI 中如何用 prompt 明确触发“保存经验/忘记经验”。

## 1. Auto-Learn 的目的

Auto-Learn 的核心目标是让 agent 在多次会话里积累**可复用经验**，减少重复说明成本，提高后续任务吞吐和一致性。

它主要解决两类问题：

1. 同一类偏好你反复说很多次（例如提交格式、沟通风格、测试习惯）。
2. 你希望 agent 在不同项目中沿用通用工程经验（例如调试步骤、常见排查顺序）。

## 2. 期望用途（建议记什么）

建议保存的内容：

- 你的稳定工作偏好（如 commit 规范、回复结构偏好）。
- 跨项目通用的工程习惯（如先测后改、先最小复现再修复）。
- 反复出现问题的通用解决套路（非项目私有细节）。

不建议保存的内容：

- 当前会话临时信息（一次性任务细节）。
- 项目私有路径、内部 API 细节、密钥/凭据。
- 未验证结论和猜测。

## 3. TUI 里如何触发“保存”

最稳定的方式是**显式表达“记住”意图**。可直接用以下句式：

- `记住：以后默认先给最小修复方案，再给重构方案。`
- `请记住我的偏好：commit message 使用 conventional commits。`
- `Remember this preference: always run affected tests after code changes.`

如果你希望归类到某个主题，也可以写：

- `记住到 debugging 主题：排查失败测试时先看最早报错，不先看最后汇总。`
- `Remember under tooling: prefer rg for search and uv run for Python scripts.`

## 4. TUI 里如何触发“忘记/删除”

最稳定的方式是**显式表达“忘记”意图**：

- `忘记我刚才关于提交前必须跑全量测试的要求。`
- `请删除这条记忆：回复必须很长。`
- `Forget this memory: always use provider X by default.`

如果你知道主题，可加主题描述提高命中：

- `在 debugging 主题里忘记这条：先重启机器再排查。`

## 5. 推荐的交互模板

你可以在 TUI 里直接复用以下模板。

保存模板：

```text
请记住这条长期偏好：
<你的偏好/经验>
如果已有类似条目，请更新而不是重复新增。
```

忘记模板：

```text
请忘记这条记忆：
<要删除的内容>
如果存在近义条目，也请一起清理。
```

校验模板（确认是否生效）：

```text
请简要复述你当前记住的与 <主题> 相关的关键经验。
```

## 6. 与 `/learn` 命令配合

建议把自然语言触发与 `/learn` 命令结合使用：

- `/learn list`：查看有哪些主题文件。
- `/learn show <topic>`：查看某主题内容。
- `/learn clean`：交互清理过时条目。
- `/learn off`：临时关闭自动学习写入。
- `/learn on`：重新开启。

推荐流程：

1. 用自然语言“记住/忘记”触发。
2. 用 `/learn show` 或 `/learn list` 快速复核。
3. 定期用 `/learn clean` 清理低质量或过时经验。

## 7. 实用建议

- 让记忆“短、稳、可执行”：一句话表达一个规则。
- 优先写“做法”，少写“背景故事”。
- 涉及安全信息时，不要用“记住”提示词。
